  • User types into cells

  • On word completion (or Enter), validate against solution

  • Apply a CSS animation class to blink red/green

No server is required unless you want to save progress (user resumes later). If you later want saving:

  • simplest: browser localStorage inside the embedded page (works in most cases, but iframe contexts can vary by browser/settings)

  • robust: store progress in Google Sheets / Firebase / a small API

What I recommend for you (Google Sites + many crosswords)

Use GitHub Pages (or Netlify) and keep each crossword as one HTML file. Then you only do this per puzzle:

  1. duplicate the file

  2. replace the grid + clues data block

  3. publish

  4. embed URL in Google Sites

This avoids H5P’s “data entry tax” entirely.
